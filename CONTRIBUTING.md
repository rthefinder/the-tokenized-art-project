# Contributing to The Tokenized Art Project

Thank you for your interest in contributing to $ART! We welcome contributions from the community.

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Help others learn and grow
- Prioritize artist empowerment in all decisions

## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in [Issues](https://github.com/rthefinder/the-tokenized-art-project/issues)
2. If not, create a new issue with:
   - Clear title and description
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details (OS, Node version, etc.)

### Suggesting Features

1. Check existing issues and discussions
2. Create a new issue with:
   - Clear use case
   - Proposed solution
   - Impact on artists and collectors
   - Consider alignment with core principles

### Pull Requests

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes following our coding standards
4. Add or update tests as needed
5. Ensure all tests pass
6. Update documentation if needed
7. Submit a PR with a clear description

### Coding Standards

- **TypeScript**: Use strict mode, proper typing
- **Solidity**: Follow best practices, add NatSpec comments
- **Testing**: Maintain >80% coverage for contracts
- **Commits**: Use conventional commits (feat:, fix:, docs:, etc.)
- **Formatting**: Run `pnpm format` before committing

### Development Setup

```bash
# Install dependencies
pnpm install

# Run tests
pnpm test

# Check linting
pnpm lint

# Format code
pnpm format
```
